What is Git?

git is an open sourse version control symstem that allows developers tracks versions of their code code overtime.

git tracks our codes by creating snaphots of the code and and can be revisited whenever necessary

Basic commands

git init is used to inttialize git in your local repository.

git status is used to check the activities of git in your local repository.

git add "filename" is used to track a single file in git.

git add.is used to track multiple files in git.

git rm --cached "filename" is used to untrack a file in git.

git commit -m "the commit message" is used when we want to save a snapshot of our code. It is like a caption or the descrition of the changes made to our codes.

git remote add origin "repo url" is used to connect your local repo to github.

git push -u origin master is used to push our code from the local repo to the remote repo.



What is Github?

Github is a web based git repository hosting service.

It is a tool that enables collaboration by hosting shared git repositories that team of developers can all contribute to.

While GitHub uses git the functionality it provides is entirely differnt from Git.

Git is a Version Control System while github is an online platform for hosting and sharing code.

Why Use GitHub?

  1. GitHub provides a great way to store your code in a safe and remote location.
  2. Many large open source project are hosted on Github.
  3. Github encourages collaboration between a team to developers.
